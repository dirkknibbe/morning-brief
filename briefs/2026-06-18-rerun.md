🛠️ *Meta-Agents & Harnesses — the deeper cut*
• *Relaymux* (Show HN) — tmux-backed meta-harness for local coding agents. The thesis: *don't hide agent runs in a black box.* Telegram is the remote control; each agent gets a visible, attach-able tmux window you can watch/interrupt/debug. Wraps `pi`/`codex`/`claude`. **Note for you: it's the exact Telegram-bot-as-control-plane pattern this brief pipeline already uses.**
• *Agentspace* — the opposite design philosophy: long-running "YOLO" Claude Code agents, each in its own disposable Docker volume + throwaway container. Full workspace isolation, detach/reattach like a multiplexer. Wraps Claude Code only (for now).
• The contrast is the story: *visibility (tmux, recoverable, local)* vs *isolation (Docker, disposable, safe)*. Both are betting the outer harness — not the model — is where autonomous coding gets reliable.

😤 *Developer Friction*
• Same root cause both harnesses dance around: autonomous agents need a *durable session* that survives the terminal closing and a *safe blast radius*. Nobody's nailed both at once — Relaymux picks visibility, Agentspace picks isolation.

💰 *Monetization Patterns*
• Both ship as `curl | bash` / `npm i -g` open-source CLIs with zero pricing — classic adoption-first wedge. The paid layer (hosted control plane, team dashboards) is conspicuously absent and up for grabs.

💡 *Opportunity Spark*
